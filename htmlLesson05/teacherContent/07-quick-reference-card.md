## KEY TALKING POINTS

### Data Types Made Simple

**TEXT** → "Names, comments - can't do math with it"
**NUMBER** → "Ratings, quantities - can calculate averages"
**CATEGORY** → "Fixed choices - super clean data, easy to count"
**DATE** → "Use date pickers, not text boxes"
**BOOLEAN** → "Yes/no questions - checkboxes"

### The Money Quote

> "Professional data analysts spend TONS of time cleaning messy data. Good form design PREVENTS that mess."

### Connecting to Future

> "In a few weeks, YOU'LL write the HTML that creates these forms. Then you'll build databases to store the data. Then you'll write SQL to analyze it. Today is your introduction to the whole pipeline."

## POTENTIAL STUDENT QUESTIONS & ANSWERS

**Q: "Why can't we just use text boxes for everything?"**
A: "Great question! Text boxes give messy data. If I ask for age and get '25', 'twenty-five', 'mid-twenties'... I can't calculate an average. Categories and numbers give clean data."

**Q: "What's the difference between radio buttons and checkboxes?"**
A: "Radio = pick ONE (like choosing a shirt size). Checkboxes = pick MANY (like choosing pizza toppings). Different data types for different needs."

**Q: "Can't JavaScript clean messy data?"**
A: "Sure, but why make extra work? It's way better to collect clean data from the start. That's what good form design does."

**Q: "What if someone wants to pick a restaurant not on our list?"**
A: "Good point! We could add 'Other' as an option. But then we get messy text data for those responses. It's always a trade-off between flexibility and clean data."

**Q: "Does HTML really work like Google Forms?"**
A: "Yes! The code looks different, but the thinking is exactly the same. You're choosing input types that match data types. Google Forms just makes it visual instead of code."

**Q: "Where does the data actually go?"**
A: "Great question! It goes to a database - usually a table that looks just like a spreadsheet. Each form submission becomes a row. Each question becomes a column. That's what you'll learn in Database Applications!"

## IF TECHNOLOGY FAILS

**Google Form won't load?**
→ Use pre-made sample dataset, focus on analysis

**Can't share form link?**
→ Fill out at teacher computer with student input

**Sheets crashes?**
→ Switch to Excel, use pre-exported data

**Projector dies?**
→ Paper form planning activity, discuss data types

## Part 1: Requirements Checklist

Review each requirement from the project specification. Check the box if you've completed it.

### HTML Structure & Semantic Markup
- [ ] Website has 4+ pages (Home, About, Services/Portfolio, Contact)
- [ ] All pages have valid HTML5 with proper DOCTYPE, html, head, and body tags
- [ ] All pages have `<meta charset="UTF-8">` and viewport meta tag
- [ ] Semantic HTML5 elements used (`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`)
- [ ] Proper heading hierarchy (one `<h1>` per page, correct `<h2>` and `<h3>` nesting)
- [ ] All pages linked via working navigation menu

### CSS & Styling
- [ ] External CSS stylesheet (separate `.css` file)
- [ ] Custom CSS written (not just Bootstrap or framework defaults)
- [ ] 3+ colors used consistently throughout website
- [ ] 2 different fonts used (one for headings, one for body text)
- [ ] Google Fonts imported and applied correctly
- [ ] Professional, cohesive visual design

### Responsive Design
- [ ] Website works on mobile (<600px)
- [ ] Website works on tablet (600px-1024px)
- [ ] Website works on desktop (1024px+)
- [ ] CSS media queries implemented OR Bootstrap Grid used (or both)
- [ ] Navigation adapts to smaller screens
- [ ] Images scale responsively
- [ ] Tested on actual mobile device or with DevTools device emulation

### Forms & JavaScript
- [ ] Contact form created with 3+ input fields
- [ ] Form has appropriate input types (text, email, tel, textarea, etc.)
- [ ] JavaScript file linked (`<script src="script.js">`)
- [ ] Form validation implemented in JavaScript:
  - [ ] Required fields checked (not empty)
  - [ ] Email format validated
  - [ ] At least 1 additional validation rule
- [ ] User receives feedback when form is submitted (success or error message)
- [ ] Form submission prevented if validation fails
- [ ] At least 1 other interactive feature (smooth scrolling, toggle menu, gallery, modal, etc.)

### Content & Media
- [ ] At least one data table (pricing, menu, schedule, etc.)
- [ ] Table has clear headers and 4+ columns
- [ ] Table has 5+ rows of data
- [ ] 3-4 images throughout website
- [ ] All images have descriptive `alt` text
- [ ] All images are optimized (reasonable file size)
- [ ] Website has original, meaningful content (no Lorem Ipsum)

### SEO & Accessibility
- [ ] All pages have descriptive title tags (<60 characters)
- [ ] All pages have meta descriptions (100-160 characters)
- [ ] Heading hierarchy is correct (h1, h2, h3 in proper order)
- [ ] Color contrast is sufficient (readable text)
- [ ] Keyboard navigation works (Tab key moves through links/buttons)
- [ ] All images have alt text
- [ ] No accessibility warnings in Lighthouse audit

### Validation & Testing
- [ ] HTML passes W3C Validator (https://validator.w3.org/) with 0 errors
- [ ] CSS passes W3C Validator (https://jigsaw.w3.org/css-validator/) with 0 errors
- [ ] Tested in at least 2 different browsers (Chrome, Firefox, Safari, Edge, etc.)
- [ ] All links work (no 404 errors)
- [ ] Form validation works correctly
- [ ] Lighthouse audit score: 80+

### Deployment
- [ ] GitHub repository created (public)
- [ ] All files pushed to GitHub (HTML, CSS, JS, images)
- [ ] README.md file created with project description and live link
- [ ] Website deployed to GitHub Pages (live and accessible)
- [ ] Repository has meaningful commit messages
